Runtime

发表于 2016-07-01   |   分类于 Object-C

故事要从一次崩溃开始…

由于你的粗心大意,调用了一个只声明没有实现的方法,结果你肯定知道了.是的,程序崩溃了.如果你打了全局断点,很容易找到问题出在哪了.但是,我们的讨论不止于此,我们要深入方法调用的背后,看看这期间到底发生了哪些事情.

Runtime

正如你所知道的 Object-C 是一门动态语言,所以只有在运行时它才会做一些决定性工作.这也就是说为什么在编译阶段,你调用了一个没有声明的方法,编译器不会报红.对于这一点,C语言恰恰是相反的,他会在编译阶段就告诉你,你调用了一个没有实现过的方法.

既然如此, Object-C 不仅仅需要一个编译器,而且也需要一个运行时系统来动态得创建类和对象、进行消息传递和转发.这个运行时系统就是我们今天的主人公 Runtime .

Runtime 是用C和汇编写的,而且是开源的,点击这里可以下载到它.

消息传递

当我们在屏幕上敲出[self doSomething]这一行代码时,编译器会对 Objective-C 代码做分析检查, 并把它翻译成一个C的函数调用,如下

  • 检测self是不是nil,如果是则会直接忽略.
  • 方法的调用者会通过 isa 指针来找到其所属的类，然后在 cache 或者 methodLists 中查找该方法，找得到就跳到对应的方法去执行.
  • 如果class中没有找到对应的方法,继续往它的superclass中找,如果还没找到会继续往上找,直到找到NSObject类为止.(如下图)
  • 一旦找到对应的函数,就去执行它的实现IMP.如果没有找到就会进入消息转发.

图片来自objc_explain_Classes_and_metaclasses

动态方法解析和转发

在上面的消息传递中,如果一直没有找到对应的函数,没有其他设置的话,就是我们在一开始提到的从崩溃开始,我们的程序会崩溃.但是在程序崩溃前, Runtime 会给我们三次拯救程序的机会:

1.动态方法解析

2.备用接收者(重定向)

3.消息转发

下面我们分别来看看这三次机会

动态方法解析

在上面的消息传递中,如果一直没有找到对应的方法, Runtime 会发送 + resolveInstanceMethod: 或者 + resolveClassMethod: 尝试去 resolve 这个消息.调用class_addMethod添加方法的实现.

备用接收者

如果 resolve 方法返回 NO, Runtime 就发送- forwardingTargetForSelector: 允许你把这个消息转发给另一个对象.切记不能返回self,这样会进入死循环状态.

完整转发

到这一步,是 Runtime 给我们最后一次拯救世界的机会.如果没有新的目标对象返回, Runtime 就会发送 - methodSignatureForSelector: 和 - forwardInvocation: 消息.

首先调用methodSignatureForSelector:方法,尝试获得一个方法签名.如果获取不到,则直接调用doesNotRecognizeSelector抛出异常.

否则就会调用forwardInvocation:方法,将上一步获取到的方法签名包装成Invocation传入.调用invokeWithTarget:手动转发消息.

Runtime 实例应用

了解了 Runtime 的一些特性,我们就可以根据它的特性做一些很magic的事情,比如:

1.Method Swizzling
2.AOP - Aspect
3.JSPatch
4.字典与模型的便捷转换
5.便捷归档与接档
6.Associated Objects

深入理解Block

发表于 2016-04-10   |   分类于 Object-C

Block概要

什么是Block

Block objects are a C-level syntactic and runtime feature. They are similar to standard C functions, but in addition to executable code they may also contain variable bindings to automatic (stack) or managed (heap) memory. A block can therefore maintain a set of state (data) that it can use to impact behavior when executed.

怎么理解这段话? 可以分成两段来理解:

  • Block 对象是一个C语言级别的语法和运行时特征: Block的形式跟C语言中函数指针的形式基本相似(后文会有讲到)
  • 包含stack或者heap中的内存的变量: Block带有自动变量

所以, 可以用一句话来表示Block:带有自动变量(局部变量)的匿名函数.
